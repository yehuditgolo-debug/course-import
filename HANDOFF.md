## 1. מה זה הפרויקט

מערכת תוכן ("Content System") עבור המותג של **שי** — תזונה + תנועה לגברים
חרדים, מטרייה: **"המינימום שמחזיק"**. האסטרטגיה מבוססת על קורס **גפן הלוי**.

המיקוד הנוכחי: **פוסט הליבה** — יועץ שמכיר את השיטה של שי, ולולאת דאטה ללמידה.
מערכת השכפול/רינדור/לוח כבר נבנתה ומשמשת כ"מפעל" שמקבל פוסט ליבה וממשיך משם.

המקור לכל זה: השיחה התחילה מניתוח 96 צילומי מסך של וובינר שיווק (Nataliya Rey /
"The Next Level") על בניית "סיסטמת תוכן" מבוססת AI. שכפלנו את הרעיון למותג של שי.

## 2. המותג והשיטה — תמצית (המקור המלא ב-method/)

**מוכרים את הרצון, לא את השיטה** (עיקרון מגפן). שי מוכר את היכולת לתפקד בלי
להשתעבד — אנרגיה, גב שלא כואב, החליפה שנסגרת — ומוכיח שזה אפשרי כי הוא חי את זה.

**3 המסרים (עמודי תוכן):**
1. אפשר בלי מהפכה — "אני אוכל המבורגר, שווארמה, פיצה. וזה עובד."
2. דיאטות נכשלות מהסבל — "אין אצלי משקל. אפס."
3. אני חי את זה — "הצלחת שלי, לא של לקוח. והאימונים שלי לא יותר משעה."

**קווים אדומים (אסור):** מספרי גוף (קילו/קלוריות/מידות), מושגים מקצועיים
(גירעון/מאקרו), הבטחות קסם ודדליינים, שפת ראווה (קוביות/חיטוב), והפיכת "הדלת"
(כרס/חליפה) ל"יעד" מדיד.

**מבחן הזהב (6 שאלות):** מדבר אל הרצון? בגובה העיניים? נתלה על אחד מ-3 המסרים?
מראה מעשה? מחזק את "המינימום שמחזיק"? לא נשמע כקסם/מספר? — אחרת החוצה.

## 3. הסטטוס הטכני — מה נבנה (Node, ESM, אפס תלויות runtime)

| רכיב | קובץ | תפקיד |
|---|---|---|
| סכמה + מכונת מצבים | `src/schema.js` | 5 פורמטים, מסע סטטוסים, makeCorePost (כולל tags+metrics) |
| מאגר | `src/store.js` | JSON store + מעברי סטטוס חוקיים |
| מנוע שכפול | `src/engine/repurpose.js` | פוסט ליבה → 5 פורמטים; suggestSchedule (פיזור כל יומיים) |
| גזירת copy | `src/engine/formats.js` | היגיון "עורך" לוקאלי לכל פורמט |
| עורך AI (אופ׳) | `src/engine/ai.js` | Claude אם יש ANTHROPIC_API_KEY; אחרת fallback לוקאלי |
| רינדור | `src/render/templates.js`, `render.js` | HTML/CSS לכל פורמט (RTL/LTR, פונט Heebo, טורקיז+זהב) → `output/` |
| PNG (אופ׳) | `scripts/render-png.js` | ייצוא HTML→PNG דרך Playwright |
| אינטגרציות | `src/integrations/index.js` | Airtable/Notion/GHL — stubs שמתעוררים עם env |
| דאשבורד | `src/server/` | לשונית תוכן (pipeline+לוח) + לשונית סוכנים |
| סוכנים | `src/agents.js` | מרשם A1–A6, לוג ריצות, סטטוסים |
| בסיס ידע | `method/` | המתודה, האסטרטגיה, גפן, voice, hooks, pillars |
| תוכנית | `COPILOT.md` | שני המנועים: יועץ + לולאת דאטה |
| הוראות קבועות | `CLAUDE.md` | פרוטוקול היועץ + קווים אדומים |
| נתוני דוגמה | `src/seed.js` | 2 פוסטי ליבה אמיתיים של שי |

**מודל הנתונים** (פוסט ליבה): `id, lang, hook1, hook2, body, cta,
tags{pillar,hookType,angle,ctaType}, metrics{reach,views,saves,shares,comments,leads,...},
formats[]`. כל פורמט: `id, type, status, scheduleDate, copy, render`.

**מסע הסטטוסים (ה"פרוטוקול" שמתאם הכל):**
`to_edit → editing → review → approved → scheduled → published`

**הרצה:**
```bash
node src/cli.js seed      # נתוני דוגמה
node src/cli.js serve     # http://localhost:3000  (לשוניות: תוכן / סוכנים)
# CLI נוסף: create / repurpose / render / schedule / list
```

**הסוכנים בדאשבורד:** A1 משכפל · A2 עורך יומי · A3 מתזמן — **חיים ולחיצים**.
A4 יועץ פוסט ליבה · A5 לולאת דאטה · A6 מפרסם GHL — **מתוכננים** (הצעד הבא).
